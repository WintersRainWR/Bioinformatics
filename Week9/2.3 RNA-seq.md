## 一、什么是Multiple test correction? 并解释 q value(很多时候也叫FDR) 和 p value 的差别

多重检验校正（Multiple test correction）是指在同时进行大量统计检验（如 RNA-seq 中对成千上万个基因做差异分析）时，对显著性结果进行调整，以避免因检验次数过多而产生大量假阳性的问题。如果不进行校正，即使所有基因都没有真实差异，也会有一部分因为随机波动而被错误判为显著。

p value 表示在零假设成立时观察到当前或更极端结果的概率，它只针对单个检验，不考虑总共做了多少次检验；而 q value（通常也称为 FDR）是在多重检验背景下的指标，表示在所有被判定为显著的结果中，假阳性所占的期望比例。简单来说，p value 关注单个基因是否显著，而 q value 则控制整体结果的错误率，更适合用于高通量数据分析中的显著性判断。

## 二、请结合上课时所讲的知识阐述DESeq2和edgeR中如何对数据进行 normalization，列出并解释具体的公式 。

在 RNA-seq 数据分析中，由于不同样本的测序深度和组成偏差不同，需要对原始 counts 数据进行归一化处理。DESeq2 主要采用基于 size factor 的方法，其思想与 rlog（relative log expression）中的步骤是一致的。具体来说，首先对每个基因计算在所有样本中的几何平均值：

$$
g_i = \left(\prod_{j=1}^{n} K_{ij}\right)^{1/n}
$$

其中 $K_{ij}$ 表示基因 $i$ 在样本 $j$ 中的 counts；然后将每个基因在各样本中的表达量除以该几何平均值得到比值：

$$
r_{ij} = \frac{K_{ij}}{g_i}
$$

并对每个样本取所有基因比值的中位数作为该样本的标准化因子：

$$
C_j = \mathrm{median}_i (r_{ij})
$$

最后得到归一化后的表达量：

$$
K_{ij}^{(\mathrm{norm})} = \frac{K_{ij}}{C_j}
$$

该方法的核心假设是大多数基因在不同样本之间不发生差异表达，因此中位数可以作为稳定的尺度估计。在此基础上，DESeq2 还提供 rlog 和 VST（variance stabilizing transformation）等变换方法，其中 rlog 通过对数尺度建模减少低表达基因的方差，而 VST 则通过拟合均值与方差之间的关系，使变换后的数据具有近似恒定方差，从而更适合下游分析。

edgeR 则采用 TMM（trimmed mean of M-values）方法进行归一化，其核心思想是利用大多数不差异表达的基因作为参考来校正样本间的组成偏差。具体而言，首先对原始 counts 按测序深度进行标准化，记为 $K_{gj}/D_j$，其中 $D_j$ 为样本 $j$ 的 library size，并选择一个参考样本 $r$。随后对每个基因计算：

$$
M_g(j,r) = \log_2\left(\frac{K_{gj}/D_j}{K_{gr}/D_r}\right)
$$

$$
A_g(j,r) = \frac{1}{2} \left[ \log_2\left(\frac{K_{gj}}{D_j}\right) + \log_2\left(\frac{K_{gr}}{D_r}\right) \right]
$$

为了减少极端值的影响，去除 $M$ 值两端约 30\% 和 $A$ 值两端约 5\% 的基因，仅保留中间稳定部分；然后对剩余基因计算权重：

$$
w_g(j,r) = \left( \frac{D_j - K_{gj}}{D_j K_{gj}} + \frac{D_r - K_{gr}}{D_r K_{gr}} \right)^{-1}
$$

并对 $M$ 值进行加权平均，得到标准化因子：

$$
C_j = 2^{\frac{\sum w_g(j,r) M_g(j,r)}{\sum w_g(j,r)}}
$$

最后将每个样本的表达量除以对应的 $C_j$ 完成归一化。相比之下，TMM 方法能够有效消除高表达基因带来的组成偏差，因此在样本差异较大时更为稳健。

## 三、利用我们以上介绍的方法和数据，分别使用DESeq2和edgeR找出uvr8突变型（uvr8）在光照前后的差异基因，保存为文本文件

DeSeq2: 输入代码

```
counts <- read.table("count_exon.txt",
                     sep = "\t",
                     header = TRUE,
                     row.names = 1)

sample_names <- c("UD1_1", "UD1_2", "UD1_3",
                  "UD0_1", "UD0_2", "UD0_3")
counts <- counts[, sample_names]
counts_filtered <- counts[rowMeans(counts) > 5, ]
condition <- factor(c(rep("Treatment", 3),
                      rep("Control", 3)),
                    levels = c("Control", "Treatment"))
col_data <- data.frame(
  row.names = colnames(counts_filtered),
  condition = condition
)
library(DESeq2)
dds <- DESeqDataSetFromMatrix(countData = counts_filtered,
                             colData = col_data,
                             design = ~ condition)
dds <- DESeq(dds)
res <- results(dds)
res_df <- as.data.frame(res)
deg <- subset(res_df,
              padj < 0.05 & abs(log2FoldChange) > 1)

write.table(deg,
            file = "DESeq2_results.txt",
            sep = "\t",
            quote = FALSE,
            col.names = NA)
```

edgeR: 输入代码

```
group <- factor(c(rep("Control", 3),
                  rep("Treatment", 3)),
                levels = c("Control", "Treatment"))

design <- model.matrix(~ group)
library(edgeR)
dge <- DGEList(counts = counts_filtered,
               group = group)
dge <- calcNormFactors(dge, method = "TMM")
dge <- estimateDisp(dge, design)
fit <- glmFit(dge, design)
lrt <- glmLRT(fit, coef = 2)
res <- topTags(lrt, n = nrow(dge))$table
deg <- subset(res,
              abs(logFC) > 1 & FDR < 0.05)
deg <- deg[order(deg$FDR), ]

write.table(deg,
            file = "edgeR_results.txt",
            sep = "\t",
            quote = FALSE,
            col.names = NA)
```

## 四、对于uvr8突变型的差异基因，定义|log2FC|>1，FDR<0.05的基因为差异表达基因。比较两个软件得到的差异基因有多少是重合的，有多少是不同的，用venn图的形式展示

代码如下：

```
counts <- read.table("count_exon.txt",
                     sep = "\t",
                     header = TRUE,
                     row.names = 1)

samples <- c("UD1_1", "UD1_2", "UD1_3",
             "UD0_1", "UD0_2", "UD0_3")
counts <- counts[, samples]
counts_filtered <- counts[rowMeans(counts) > 5, ]
group <- factor(c(rep("Control", 3),
                  rep("Treatment", 3)),
                levels = c("Control", "Treatment"))

library(DESeq2)
col_data <- data.frame(row.names = colnames(counts_filtered),
                       group = group)
dds <- DESeqDataSetFromMatrix(countData = counts_filtered,
                             colData = col_data,
                             design = ~ group)
dds <- DESeq(dds)
res_deseq <- as.data.frame(results(dds))
deg_deseq <- subset(res_deseq,
                    padj < 0.05 & abs(log2FoldChange) > 1)
write.table(deg_deseq,
            file = "DESeq2.txt",
            sep = "\t",
            quote = FALSE,
            col.names = NA)

library(edgeR)
dge <- DGEList(counts = counts_filtered,
               group = group)
dge <- calcNormFactors(dge)  # 默认就是 TMM
design <- model.matrix(~ group)
dge <- estimateDisp(dge, design)
fit <- glmFit(dge, design)
lrt <- glmLRT(fit, coef = 2)
res_edger <- topTags(lrt, n = nrow(dge))$table
deg_edger <- subset(res_edger,
                    FDR < 0.05 & abs(logFC) > 1)
write.table(deg_edger,
            file = "edgeR.txt",
            sep = "\t",
            quote = FALSE,
            col.names = NA)

library(VennDiagram)
venn_list <- list(
  DESeq2 = rownames(deg_deseq),
  edgeR  = rownames(deg_edger)
)
venn.diagram(
  x = venn_list,
  fill = c("red", "blue"),
  filename = "venn.png"
)
```

## 五、对于edgeR找出的FDR<0.05的基因，选出log2FoldChange最大的10个基因和最小的10个基因。计算原始表达量矩阵的log10CPM值并对每个基因进行Z-score处理，使用刚才筛选出来的20个基因绘制热图（heatmap）作为最后结果输出。

输入代码：

```
library(dplyr)
res_sorted <- edger %>% arrange(logFC)
top_genes <- rbind(
  head(res_sorted, 10),   # 下调
  tail(res_sorted, 10)    # 上调
)
counts_subset <- counts_filtered[rownames(top_genes), ]

calc_zscore <- function(mat){
  cpm <- t(1e6 * t(mat) / colSums(mat))
  log_mat <- log10(cpm + 1)
  z <- t(scale(t(log_mat)))   # 按行标准化
  return(z)
}
z_mat <- calc_zscore(counts_subset)
annotation_col <- data.frame(
  Group = factor(c(rep("Control", 3),
                   rep("Treatment", 3)))
)
rownames(annotation_col) <- colnames(z_mat)
annotation_row <- data.frame(
  Regulation = factor(c(rep("Down", 10),
                        rep("Up", 10)))
)
rownames(annotation_row) <- rownames(z_mat)

library(pheatmap)
pheatmap(z_mat,
         cluster_rows = FALSE,
         cluster_cols = FALSE,
         annotation_col = annotation_col,
         annotation_row = annotation_row,
         show_rownames = FALSE)
```
