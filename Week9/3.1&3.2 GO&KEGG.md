## 二、请问上面的例子中， Fold Enrichment和P value是如何计算的? 请写出公式，并解释原理。此外，在定义显著富集的 GO terms 时为什么一般不是参考P value的大小，而是要计算一个 FDR来做为参考？

在 GO 富集分析中，我们希望判断某一 GO term 在目标基因集合中是否显著富集，即其出现频率是否高于随机情况。设 $N$ 为背景基因总数，$M$ 为背景中属于某一 GO term 的基因数，$n$ 为目标基因集的大小，$k$ 为目标基因集中属于该 GO term 的基因数。Fold Enrichment 用于描述富集程度，其定义为 $\text{Fold Enrichment} = \frac{k/n}{M/N}$，其中 $\frac{k}{n}$ 表示该 GO term 在目标基因集中的比例，而 $\frac{M}{N}$ 表示其在背景中的比例，因此该值反映了目标基因集中该功能被放大的倍数。

为了评估这种富集是否具有统计学意义，通常采用超几何分布进行检验，其 p value 计算公式为 $P = \sum_{i=k}^{n} \frac{\binom{M}{i} \binom{N-M}{n-i}}{\binom{N}{n}}$。该概率表示在从背景基因中随机抽取 $n$ 个基因的情况下，观察到至少 $k$ 个属于该 GO term 的概率。如果该概率很小，则说明该 GO term 在目标基因中的出现不太可能是随机造成的，从而具有统计学意义。

然而，在实际 GO 分析中通常需要同时对大量 GO terms 进行检验，这会引入多重检验问题，从而显著增加假阳性的比例。因此，单独依赖 p value 并不能可靠地判断显著性，需要进一步进行多重检验校正。常用的方法是计算 FDR（False Discovery Rate），其本质是对 p value 进行调整，用于控制所有被判定为显著的 GO terms 中假阳性的期望比例。相比之下，p value 仅反映单个检验的显著性，而 FDR 则从整体上控制错误发现率，因此在实际分析中通常以 $FDR < 0.05$ 作为判断显著富集的标准。

## 三、请用 KEGG enrichment 分析上一章(GO enrichment analysis)中的基因集，比较两章的结果，总结两种方法得到的生物学意义有哪些异同。

GO 数据库中的基因功能注释具有明显的层级结构，其顶层分为 Molecular Function、Biological Process 和 Cellular Component 三大类，并在此基础上由粗到细逐级细化形成有向无环图（DAG）结构。GO 注释既包含人工整理的内容，也包含基于计算方法推断的自动注释。GO 富集分析的核心是从功能分类的角度出发，将基因映射到不同功能类别中，从而识别在目标基因集中显著富集的功能项，其关注点主要在于基因的功能归类，而不直接强调基因之间的相互作用关系。

相比之下，KEGG 数据库以生物通路为核心组织信息，不仅包含代谢通路，还整合了信号转导、疾病以及分子相互作用等多层面的数据。KEGG 中的基因是按照具体通路进行组织的，这些通路本质上反映了一系列基因或分子之间的相互作用关系。KEGG 注释主要来源于人工整理和实验支持，整体上具有较高的可靠性。KEGG 富集分析则侧重于从通路层面理解基因的功能，能够揭示差异基因在具体生物过程中的协同作用和调控机制。

总体而言，GO 富集分析更偏向于提供功能分类上的概览，适合描述基因“做什么”；而 KEGG 富集分析更强调通路和分子之间的相互作用，有助于解释基因“如何协同发挥作用”。两者在功能解释上具有互补性，通常需要结合使用以获得更全面的生物学意义解释。
