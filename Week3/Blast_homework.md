3> 经查阅得知：BLAST之所以远快于传统动态规划，核心在于其采用了启发式的“种子-延伸”策略，放弃了保证最优解的目标以换取速度。它首先通过哈希表为查询序列建立短词索引，并借助打分矩阵生成允许保守替换的邻接词，从而在扫描数据库时通过查表快速定位所有潜在的匹配区域，避免了动态规划中全矩阵计算的平方级复杂度。为了进一步提速，BLAST采用“两步命中”机制，要求在同一条对角线上近距离出现两个独立种子才触发延伸，有效过滤了随机噪音。随后仅对通过筛选的种子进行快速的无空位延伸，只有得分显著的区域才被施以含有空位的动态规划计算。通过这种层层过滤、将最耗时的计算严格限制在极小候选区域的分层架构，BLAST实现了对海量数据库的快速搜索。

4> PAM250矩阵之所以存在对称和不对称两种形式，根本原因在于它们代表了同一套数据在不同计算阶段和应用场景下的不同形态。不对称的PAM250矩阵是直接由突变数据统计得到的原始形式，它记录了从一种氨基酸变为另一种氨基酸的“可接受点突变”概率，由于突变在进化方向上的频率并不完全相同，因此原始矩阵在数学上是不对称的。然而，在实际的序列比对应用中，我们比较两条蛋白质序列时，并不知道哪一条是祖先、哪一条是后代，替换的方向是未知的，因此必须假设两条序列的地位是对等的。为了满足这一要求，对称的PAM250矩阵应运而生。它通常通过两种方式获得：一是将原始突变概率与氨基酸的背景频率结合，计算出一个称为“对数几率”的得分矩阵，这种数学变换会天然地产生一个对称矩阵，其数值代表了某种替换在进化相关序列中出现与随机出现之间的可能性比值；二是直接对原始不对称矩阵进行对称化处理，例如通过取几何平均等数学手段。在应用上，我们日常用于序列比对的PAM250矩阵几乎都是对称形式，因为它保证了比较的公平性。而不对称的原始形式则更多地用于理论探讨或构建更复杂的进化模型，例如在研究特定进化路径或构建有向进化距离时才会用到。

2> 编写出 $\mathrm{bash}$ 文件如下方：

```
#!/bin/bash

# ==============================
# 1. 原始蛋白序列
# ==============================
ori_pr="MSTRSVSSSSYRRMFGGPGTASRPSSSRSYVTTSTRTYSLGSALRPSTSRSLYASSPGGVYATRSSAVRL"

# ==============================
# 2. 生成10个随机序列
# ==============================
random_sequences=()

for ((j=1; j<=10; j++))
do
    si=0
    L=${#ori_pr}
    ei=$(($L-1))
    shuffled=""

    # 打乱序列
    for i in $(seq $si $ei | shuf)
    do
        shuffled=$shuffled${ori_pr:$i:1}
    done

    random_sequences+=("$shuffled")
done

# ==============================
# 3. 输出到FASTA文件
# ==============================
fasta_file="random_sequences.fasta"
> "$fasta_file"

for ((i=0; i<10; i++))
do
    echo ">seq$i" >> "$fasta_file"
    echo "${random_sequences[i]}" >> "$fasta_file"
done

# ==============================
# 4. 两两 BLAST
# ==============================
result_file="blast_results.txt"
> "$result_file"

for ((i=0; i<10; i++))
do
    for ((j=i+1; j<10; j++))
    do
        echo "Comparing seq$i vs seq$j" >> "$result_file"

        # 提取序列 i
        seq_i=$(awk "/>seq$i/{getline; print}" "$fasta_file")
        echo -e ">seq$i\n$seq_i" > query.fa

        # 提取序列 j
        seq_j=$(awk "/>seq$j/{getline; print}" "$fasta_file")
        echo -e ">seq$j\n$seq_j" > subject.fa

        # BLAST 比对
        blastp -query query.fa \
               -subject subject.fa \
               -outfmt "6 qseqid sseqid pident length mismatch gapopen evalue bitscore" \
               >> "$result_file"

        echo "" >> "$result_file"
    done
done

# 删除临时文件
rm -f query.fa subject.fa

echo "All done! Results saved in $result_file"
```
